### System prompt additions made to `chatController.ts`:

- **`## Business Rules & Policies`** — Full minimum spend table, appetizer-only event support, drink ticket policy, AV policy, decorations/vendors/corkage/cancellation/invoicing rules.
- **Choice menu pricing clarity** — Each guest chooses one item per course; GF base +$3.90; add-ons extra.
- **Deposit timing** — Due on confirmation; team handles payment details.
- **Severe allergy protocol** — Flag as `"SEVERE ALLERGY: [type]"` in specialRequests and tell the guest the kitchen team will follow up.
- **After inquiry submission** — Team follows up within 24 hours.
- **`## Menu Style`** — Bot asks about Reduced Menu / Set Course / Buffet before building menu. Drink tickets asked at same time. Reduced Menu template defined.
- **`## Recommendation Scenarios`** expanded — Corporate lunch-and-learn, appetizer-only cocktail events, wedding guest dinners, event planners, budget-conscious guests, group size vs. room tradeoff (20–50 Fri/Sat), out-of-town guests, customisation beyond standard.
- **`## Ambiance & Atmosphere`** (new) — Dress code (smart casual to business casual), per-location atmosphere descriptions, décor/vendor policy proactively volunteered when asked.
- **`## After Conversation`** (new) — What happens after submission, payment/deposit question handling, cancellation question handling, how to handle post-submission change requests.
- **Current date injection** — `now.toLocaleDateString("en-CA", ...)` injected at top of every prompt to prevent date hallucination.
- **Verbosity rules** — 2–4 sentences max, no bullet lists, no bold text mid-sentence.

## 4. OpenTable Availability Integration (Apify)

Real-time availability checking via Apify's OpenTable Booker actor ($0.05/check). No official OpenTable partner API required.

### New files:

- **`backend/src/controllers/availabilityController.ts`** — Proxies to Apify REST API. Accepts `?date=YYYY-MM-DD&partySize=N&time=HH:MM`. 12-second timeout with clean fallback. Returns `{ available, slots, date }`.
- **`backend/src/routes/availabilityRoutes.ts`** — `GET /api/availability`.

### Modified files:

- **`backend/src/app.ts`** — Mounted `/api/availability` with `availabilityLimiter` (30 req/min per IP).
- **`backend/src/config/env.ts`** — Added `apifyToken` and `opentableRestaurantId`.
- **`backend/src/controllers/chatController.ts`** — Added `CHECK_AVAILABILITY_TOOL`; handler calls `/api/availability` internally; conversation order step 4 triggers the check after room recommendation; graceful fallback if service is unconfigured.

### Owner action required:

Before availability checking will work, add to `backend/.env`:

```
APIFY_TOKEN=apify_api_...
OPENTABLE_RESTAURANT_ID=<toscana_opentable_id>
```

1. Create Apify account at apify.com → Account → Integrations → API token
2. Find your OpenTable restaurant ID from your OpenTable restaurant manager portal or from the URL on your public OpenTable listing page

The bot falls back to "The team will confirm availability within 24 hours" if these are not set.
